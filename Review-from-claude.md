## 三、算法诊断：为什么只有 7 个因子

### 3.1 可达空间小得可以精确算出

生产矿机 `CategoricalE3KAN` 的硬化输出恒为 `Sub(原子A, 原子B)`（`e3.py:330`）。四个 profile 的原子库与可达硬公式空间：

| Profile | 原子构成 | 原子数 | 有序对空间 |
|---|---|---:|---:|
| short_price | 4窗×4价格×2族 | 32 | 992 |
| long_price | 4窗×4价格×2族 | 32 | 992 |
| reversal | 5窗×4价格×2族 | 40 | 1,560 |
| price_volume | 28+7+7 | 42 | 1,722 |
| **合计** | | | **≈ 5,266** |

**整个因子宇宙约五千个"双原子差"。** 7 个不是挖矿失败，而是这个空间的真实产能就是个位数。

### 3.2 漏斗解剖（v5：256 → 7，产率 2.7%）

- **122 重复（47.7%）是收敛性重复**，不是生日碰撞（64 抽样对 ~1-1.7k 空间的随机碰撞期望仅 ~2 次）：256 个矿机优化同一目标同一数据，attempt 间差异只有 logits 初始化噪声（0.01×randn）与 20 日块 bootstrap（`e3_runner.py:226,400`），大部分种子滑向同一批 argmax。没有任何机制要求第 k 个矿机避开前 k−1 个的战果——这正是图反馈要解决的问题（§五）。
- **103 保真失败（40.2%）是松弛税**：训练最大化 soft 混合的 Pearson 日 IC，交付验收的是硬公式的 RankIC——连续 vs 离散、Pearson vs Spearman 双重错位。
- 关键事实：**当前空间可闭式穷举**。逐日计算原子协方差矩阵与原子—标签协方差向量，任意 `a−b` 的日截面 IC 有闭式（`cov(a−b,y)=cov(a,y)−cov(b,y)`；`var(a−b)=var(a)+var(b)−2cov(a,b)`），全部 5,266 个公式的精确全样本得分为分钟级计算。当前用 76,800 步 Adam 近似求解一个可精确解的 argmax。**可微性的价值只在空间超出可枚举规模后出现——这不否定 KAN 路线，而是指明它该去挣钱的地方。**

### 3.3 根因排序

**空间太小（决定上限）＞ 无互补机制（决定重复率）＞ 松弛税（决定产率）。** 三者都不是重跑 v6 能改变的。

## 四、算法与模型结构建议

总方针：**结构离散、参数连续、库递归增长**。可枚举的结构层用精确枚举（同时天然充当 E5 对照）；组合爆炸的结构层用可微/进化搜索；连续参数层永远用梯度——这是 KAN 在框架中不可替代的位置。

### 4.1 轴一：算子词汇扩充——以 QuantaAlpha `function_lib.py` 为词汇规范

已核查：`/zju_0012/htq/aaai26_alpha/QuantaAlpha/quantaalpha/factors/coder/function_lib.py`（988 行，约 60 算子，Alpha101/GTJA-191 语系）。**采纳方式：作为词汇规范（vocabulary spec）而非直接 import**——它是 pandas 实现，无类型契约、无因果审计、无 support 掩码语义；MIRAGE 的 DSL 有严格的类型/因果/支撑传播契约和 Pandas-Torch 双执行器奇偶校验。正确做法是逐个算子在 typed DSL 内重实现（契约 + Torch parity + 因果检查），并把 function_lib 作为 parity 测试的第三参考实现。

**分层准入清单**（按类型化难度与预期 alpha 价值排序）：

| 优先级 | 算子 | 类型化要点 |
|---|---|---|
| **T1 直接可纳** | `TS_STD`、`TS_MAX/TS_MIN`（比率形式无量纲化）、`TS_RANK`（天然无量纲）、`TS_CORR`（**最优先**——输出天然无量纲，且当前量价从未交互）、`EMA/WMA/SMA/DECAYLINEAR`（与输入同类型）、`TS_SUM`、`TS_PCTCHANGE`（≈现有 Return）、`ABS/SIGN`、`TS_ARGMAX/TS_ARGMIN/HIGHDAY/LOWDAY`（无量纲位置） | 现有五类型系统直接容纳；严格 support 语义照搬 TsMean 的满窗要求，**不采纳** function_lib 的 min_periods 宽松语义 |
| **T2 需扩类型系统** | `WHERE/GT/LT/GE/LE` 条件族（引入 boolean_ts 类型）、`COUNT/SUMIF/FILTER`、`TS_QUANTILE/PERCENTILE`、`TS_SKEW/TS_KURT`（无量纲）、`REGBETA/残差`（双序列回归，β 无量纲；滚动回归的因果窗口要严格审计） | boolean 类型是一次性投入，解锁整个条件算子族与状态切换类机制 |
| **T3 暂缓** | `MACD/RSI/BB_*` 技术指标族 | 均为 T1 算子的复合，库递归增长（§4.4）后可自动涌现，不必作为原语 |
| 横截面族 | `RANK/MEAN/STD/SKEW`（横截面） | 现有 CSRank 已是类型汇点；横截面标准化可作为最终层选项，中间层暂不开放（保持可解释性红线） |

另建议补充 function_lib 没有的 K 线结构原子：`(High−Low)/Close`、`(Close−Open)/(High−Low)`、上下影线比率——OHLC 目前只被当作四条平行价格序列，K 线内部结构完全未用。

词汇从 6 族到 ~18 族，原子库从 ~40/profile 到 200+/profile，双原子空间即达 10⁵；配合 §4.2 立即超出可枚举规模。

### 4.2 轴二：组合深度

1. **k-稀疏多边**：每极性从选 1 原子放宽为 top-k 硬化，公式形态 `(A₁+…+Aₖ)−(B₁+…+Bₖ)`，空间 C(N,k)² 组合增长——从这里开始可微门控真正优于枚举。
2. **两层组合**（提案 §6.4 中间公式向量）：第一层产 m 个中间无量纲信号，第二层再门控组合，深度 4→7-8。
3. **库递归增长（R5/DreamCoder 环，最重要）**：每轮入选因子规范化后注入下一轮原子库作为新叶子，配残差标签开下一轮。空间随库增长，因子越挖越深越多——这是"挖掘框架"区别于"筛选脚本"的本质机制，同时实现提案 §4.11 原语治理与 R5"跨任务复用压缩"准入。

### 4.3 轴三：连续参数（KAN 挣回名分）

1. **参数化原子**：可学习窗口门控（提案 §4.6 已有设计）、阈值 hinge(x−θ)、幂 |x|^p、饱和 tanh(x/τ)；梯度学参数，MDL 网格吸附硬化 + 保真审计。机制卡形状从当前的**线性 ramp**（已核实为平凡等差直线）变成真实非线性——可解释性从"结构可追溯"升级为"形状可分析"。
2. **S1b 修复后的形状约束样条**（R1 单调锥 cumsum/softplus 重参数化 / R2 先硬后软序贯分解）作为二号矿机，产出等级 2 因子，同时回答"E3 为什么叫 KAN"。
3. 样条回归后采用共享样条基底（提案 §12.2）。

### 4.4 训练与硬化修法（消灭 40% 保真税）

- 可枚举区域直接精确解（保真差恒为零）；
- 非枚举区域：自蒸馏项 `‖soft_score − hard_score‖²` 进训练目标；hard-ST 从第 0 步开始或每 T 步渐进剪枝低概率原子（DARTS 离散化漂移标准解法）；训练中在线监控 NRMSE(soft,hard) 超阈提前放弃，别把 300 步花完才在门口处死；
- **目标对齐**：准入门槛是 RankIC 就用可微软排序（soft-rank）训 Spearman 代理或混合目标。

### 4.5 库级选择：从"逐因子门槛"到"池边际价值"

- 7 因子 vs Alpha158 的 158 特征对 LightGBM 头是结构性不公平；目标库规模 **50–200**。
- 贪心前向选择按池边际价值（Δ池验证 IC / 有效秩增量，提案 §5.8/§10.8），逐因子 RankIC 下限降为卫生底线；产物顺便是最好的论文图之一：**库规模—池价值曲线**（KAN 库 vs GP/SR 库 vs Alpha158 截断子集）。
- **多保真漏斗**（§12.5）：闭式枚举/廉价筛（10⁴–10⁵ 候选）→ 全面板验证 RankIC → 只对库快照做池边际+回测；规范哈希缓存防重算。
- **多 horizon 标签**（1/5/10/20 日）分头挖，产出不同衰减速度的因子族。
- 置换对照升级到**池级**：置换标签下跑完整多轮挖掘+选库，用置换库的池回测做零假设分布。现有记账纪律照搬，规模化时不丢。

## 五、图网络：按 principal 指示升级为核心组件

> principal 2026-07-17 口头指示：图网络是符合直觉的重要组件，优先级提升，不是消融选择。
> 治理注意：当前三处代码硬编码 `graph_unlock_allowed: False`，解锁条件（S2 证明库价值 + S3 证明冗余）源自 Plan C 决议且被 §25.1 第 4 条保留。**按项目自身规则，提升图优先级需要一条正式的 principal 书面决策日志**（一段话即可：修订图解锁条件为"随协同挖掘主线并行开发，发布主张仍受 Gate B 三层对照约束"）。建议尽快补记，避免后续 agent 因权威冲突拒绝执行。

### 5.1 定位重构

图不再是"结果证明需要后才建的末位组件"，而是**协同挖掘的大脑**：§3.2 诊断的收敛性重复（47.7%）本质上就是"矿机之间没有协调"——图是这个问题的第一性解，这与 principal 的直觉一致。同时保留可证伪性：Gate B 的三层对照（随机 < 无图自适应 < 图控）**内建为同一控制器接口的四个实现**（random / flat / bandit / graph），对照不再是拖延图的理由，而是图交付时自带的证据。

### 5.2 图 v1：确定性消息传递控制器（先 work，后学习）

**图状态**（二部异构图，提案 §10.2 的 lite 实现，3 类边起步）：

- **因子节点特征**：验证 RankIC、池边际价值、相关性指纹嵌入、AST 结构特征（算子直方图、深度、字段集、窗口谱）、年龄与衰减斜率；
- **矿机节点特征**：所辖子空间描述子（profile/原子子库/窗口段）、近期接受率、新颖率、保真通过率、已耗预算;
- **边**：`produce`（矿机→因子，带 attempt 元数据）、`signal`（因子—因子，|corr| 超阈加权）、`structural`（因子—因子，规范子树 Jaccard 相似度）。

**控制器 v1 = 2–3 轮类型化消息传递的确定性聚合**（零训练、完全可审计），输出四条反馈路径：

| 反馈路径（提案 §10.3–10.6） | v1 实现 |
|---|---|
| ① 残差任务分配 | 从因子邻域的"未解释补集"给每个矿机派发残差方向（对当前库回归后的残差标签 + 邻域限制） |
| ② 多样性排斥 | 每个矿机的排斥集 = 其近期产物在 signal 边上的最近邻因子；训练目标加 λ·Σ\|corr(score, 排斥集)\| |
| ③ 结构化交叉变异 | structural 边上的互补性得分提名父代对；类型合法子树交换 + 变异菜单（窗口抖动/同签名算子替换/库子树嫁接）——类型系统保证合法性 |
| ④ 预算与早停 | 矿机节点聚合的边际价值 → bandit 权重分配下一轮 attempt；连续低产矿机降权/重划子空间 |

## 一、总诊断（架构级）：公证机器与科研自律的错配

两种威胁模型被焊死在同一套机器里：

## 二、论文目标不变前提下的证据链倒推

### 2.1 可解释性主张目前是平凡真（vacuously true）

本评审核实：v6 入库 7 因子全部为教科书级已知族的双原子差——`Return(Low,2) − VolumeChange(Volume,60)`（短反转−量能异动）、`TsMean(Close,20) − Close`（均值回归）、量 z-score 类。两个后果：

1. 假设空间被硬约束为双原子差 → 公式**按构造**可读。评审一句话击穿："你把空间限制到只能产生平凡公式再声称产物可解释——这是空间的定义，不是方法的成果。"
2. 因子全是已知族 → "可解释"退化为"可辨认"；盲评高通过率在此复杂度下**不携带信息**。

**推论（本评审核心观点之一）：可解释性主张的非平凡性与性能非劣效收敛于同一瓶颈——空间规模。** 只有公式深到"需要机制卡才读得懂"，盲评通过率才开始度量方法而非度量空间。词汇扩张 + 组合深度（M-B）同时是两个 headline 主张的成立前提。论文应报告**公式复杂度分布 × 盲评通过率**联合分布："在复杂度 X 处仍保持通过率 Y"才是可解释性交付的证据形态。当前 7 因子的正确 framing 是"矿机在小空间重新发现已知族的 sanity check"（加分项），不是产品（减分项）。

### 2.2 已冻结的非劣效判据在结构上不可赢 → v9 判据重设计【需 principal 决策日志】
