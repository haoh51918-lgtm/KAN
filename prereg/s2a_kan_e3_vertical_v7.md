## Exact source-mining rebind

v7 does not mine again and must not issue a new mining entitlement. It uses the
entire immutable v6 mining topology only after a
`mirage_mining_rebind_receipt_v1` verifies:

- v6 base lock, config, implementation lock, topology preclaim, label
  entitlement, and first authority receipt;
- the top bundle and all six children, including every manifest and payload;
- unchanged topology, paths, ordered KAN lineage, factor identities, blackbox
  pairing, tensors, trajectories, seeds, bootstrap, and AST replay;
- equality of every mining-affecting scientific configuration field;
- v7 base and implementation identities;
- absence of v6 development opening and any v7 mining preclaim or entitlement.

The source is read in place. Copying, mutation, reselection, reordering,
threshold changes, and parameter changes are forbidden. Rebind validation is
data-independent and must not read labels.

## Software correction

The matched-MLP checkpoint replay is projected to the published sparse raw PIT
index before the existing zero-tolerance equality comparison. This reproduces
publication semantics exactly. It performs no imputation, interpolation,
cropping, tolerance relaxation, or prediction change. A value mismatch after
projection remains a hard failure.
