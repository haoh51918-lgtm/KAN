# 🧭 MIRAGE-KAN 阶段检查点：S2a v2 已重新预注册

## ✅ 一句话结论

项目已停止旧 Plan C 的延伸路线，并按最新版 proposal 第 25 节重新冻结为 **KAN-centered S2a v2**：纯符号 KAN 是唯一生产矿机，GP/SR 和 MLP 只做证伪对照，任何对照产物都不能进入正式因子库。

本次仅完成治理修正与实验设计锁定，**尚未运行 v2 训练、挖掘或 2022–2025 回测**。

## 🔎 为什么必须重建 v2

`KAN_Alpha_PR.md` 在旧 v1 挖掘进程运行期间更新。新 proposal 明确废止“让 KAN 去中心化”的 Plan C，并要求：

- KAN 可解释性是首要主张；
- 纯符号 E3 KAN 立即作为 S2 主力矿机；
- MLP 只能作为容量匹配的黑盒对照；
- typed GP/SR 只能作为离散搜索对照；
- 主判据从“必须大幅超越”调整为“性能非劣效 + 可解释性交付”。

proposal 是唯一权威，因此旧路线不能继续修补后执行。

## 🧠 v2 的真实 KAN 边界

| 模块 | 角色 | 是否可进入主因子库 |
|---|---|:---:|
| E3 纯符号 KAN | 可微门控学习原语、变量与窗口，并硬化为固定 AST 公式 | ✅ |
| typed GP/SR | 同 DSL、同预算的离散搜索对照 | ❌ |
| 容量匹配 MLP | 检验 KAN 是否为解释性付出明显性能代价 | ❌ |
| 标签置换 KAN | 检查搜索选择假阳性 | ❌ |
| Alpha158 | 真实 Quanta baseline 回放 | ❌ |

AST 是“抽象语法树”，即公式的机器可执行结构；E3 表示没有自由 B 样条残差的纯符号 KAN。主库中的每个因子必须由真实 KAN 门控训练产生，不能把 GP 公式交给 KAN 执行一次后改名。

## 🧪 冻结的最小完整链路

1. 4 个异质 profile × 64 个独立 KAN 矿机，共 256 次真实训练尝试；
2. 每个矿机只硬化一次、最多产出一个等级 1 公式；
3. 主库目标 8–16 个因子，至少覆盖 3 个 profile；
4. 同预算运行 GP/SR、完整 KAN 标签置换、容量匹配 MLP；
5. 所有产物冻结后，只打开一次 2022–2025 development backtest；
6. 五个实验臂统一进入当前 Quanta + LightGBM + TopkDropout + 成本模型；
7. 计算机器判决、机制卡、综合报告，并决定是否进入 formal S2。

## 📏 主要判据

主指标是净 Information Ratio（IR，信息比率）的配对 20 交易日块 bootstrap：

- `KAN − Alpha158` 的单侧 95% 下界必须不低于 `-0.022`；
- RankIC（每日横截面排序相关）差不低于 `-0.003`；
- 最大回撤差不低于 `-0.03`；
- 平均换手和实付成本不超过 Alpha158 的 `1.20` 倍；
- 至少 4 年中的 3 年主动收益不差于 Alpha158；
- 真实效力阈值下的标签置换假阳性数必须为 0。

显著超越 baseline 会作为增强证据，但不再是 proposal 成立的唯一方式。

## 🪪 可解释性边界

主库每个因子都必须有固定 AST、完整 KAN 谱系、硬化收据、soft–hard 忠实度、变量/滞后干预和机制卡。开发回测可以在这些机器门槛完成后运行；但在至少两位真实人类量化评审完成匿名盲评前，库只能称为“计算透明的候选库”，不能宣称为“正式可解释因子库”。
